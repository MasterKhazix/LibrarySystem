# 图书管理系统 — 傻瓜式开发流程规划

> **项目配置确认**（基于 `pom.xml`）：
> - JDK：**21**
> - Jakarta Servlet API：**6.0.0**
> - MySQL 驱动：**8.3.0**（`mysql-connector-j`）
> - 基础包名：`com.qingshan.librarysystem`
> - 打包方式：**war**
> - 技术栈：原生 **Servlet + JSP + JDBC**（无 Spring、无 MyBatis）

## 一、整体架构一览（先搞清楚要建什么）

```
浏览器 (Chrome)
    │
    ▼
┌──────────────────────────────────────┐
│  Tomcat 10+ (Jakarta EE)             │
│                                      │
│  ┌──────────────────────────────────┐│
│  │  Filter (LoginFilter)            ││  ← 登录拦截
│  ├──────────────────────────────────┤│
│  │  Controller (Servlet)            ││  ← 接收请求、调 DAO、转 JSP
│  │  LoginServlet / BookServlet ...  ││
│  ├──────────────────────────────────┤│
│  │  Model (JavaBean + DAO)          ││  ← 数据封装 + JDBC 操作
│  │  User / Book / BorrowRecord      ││
│  │  UserDAO / BookDAO / ...         ││
│  ├──────────────────────────────────┤│
│  │  View (JSP)                      ││  ← 页面展示
│  │  login.jsp / book_list.jsp ...   ││
│  └──────────────────────────────────┘│
│                                      │
│  JDBC ──────► MySQL 8.0 (library_db) │
└──────────────────────────────────────┘
```

**三层对应关系**（MVC 经典模型）：

| 层 | 技术 | 职责 |
|---|---|---|
| Model | JavaBean + DAO 类 | 封装数据 + 直接用 JDBC 读写 MySQL |
| View | JSP + JSTL + EL | HTML 页面展示，接收 Servlet 传来的数据 |
| Controller | Servlet (`doGet`/`doPost`) | 接收请求参数 → 调 DAO → 转发/重定向到 JSP |

| 字段 | 类型 | 约束 | 说明 |

| 字段 | 类型 | 约束 | 说明 |

| create_time | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**建表 SQL**（完整可执行）：

```sql
-- 用户表
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL UNIQUE,
    password VARCHAR(100) NOT NULL,
    real_name VARCHAR(50) NOT NULL,
    phone VARCHAR(20) DEFAULT NULL,
    email VARCHAR(100) DEFAULT NULL,
    role ENUM('admin', 'user') NOT NULL DEFAULT 'user',
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 图书表
CREATE TABLE books (
    id INT AUTO_INCREMENT PRIMARY KEY,
    title VARCHAR(200) NOT NULL,
    author VARCHAR(100) NOT NULL,
    isbn VARCHAR(30) UNIQUE,
    publisher VARCHAR(100) DEFAULT NULL,
    category VARCHAR(50) DEFAULT NULL,
    description TEXT DEFAULT NULL,
    total_copies INT NOT NULL DEFAULT 1,
    available_copies INT NOT NULL DEFAULT 1,
    location VARCHAR(50) DEFAULT NULL,
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 借阅记录表
CREATE TABLE borrow_records (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL,
    book_id INT NOT NULL,
    borrow_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    due_date DATE NOT NULL,
    return_date TIMESTAMP NULL,
    status ENUM('borrowed', 'returned', 'overdue') DEFAULT 'borrowed',
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (book_id) REFERENCES books(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 阶段三：Util 层 — 工具类（第 8 步 ~ 第 9 步）

#### 第 8 步：创建 `DBUtil.java`（JDBC 连接工具）
路径：`util/DBUtil.java`

核心职责：提供静态方法 `getConnection()` 返回 `java.sql.Connection`。

```java
// 关键代码骨架：
public class DBUtil {
    private static final String URL = "jdbc:mysql://localhost:3306/library_db"
            + "?useSSL=false&serverTimezone=Asia/Shanghai&characterEncoding=utf8";
    private static final String USER = "root";
    private static final String PASSWORD = "你的MySQL密码"; // 改成你自己的

    static {
        try {
            Class.forName("com.mysql.cj.jdbc.Driver");
        } catch (ClassNotFoundException e) {
            e.printStackTrace();
        }
    }

    public static Connection getConnection() throws SQLException {
        return DriverManager.getConnection(URL, USER, PASSWORD);
    }

    // 关闭资源工具方法
    public static void close(Connection conn, PreparedStatement ps, ResultSet rs) { ... }
}
```

### 阶段五：Filter 层 — 登录拦截（第 13 步）

#### 第 13 步：创建 `LoginFilter.java`
路径：`filter/LoginFilter.java`

逻辑要点：
1. 实现 `jakarta.servlet.Filter` 接口
2. 用 `@WebFilter("/*")` 注解拦截所有请求
3. 在 `doFilter` 中：
   - 放行规则：请求路径包含 `/login`、`/register`、`/css/`、`/js/`、`/index.jsp` → 直接 `chain.doFilter()`
   - 拦截规则：其他所有路径 → 检查 `session.getAttribute("user")`，为 null 则重定向到 `login.jsp`

### 阶段六：Servlet 层 — 控制器（第 14 步 ~ 第 19 步）

> **核心开发原则**：每个 Servlet 用 `@WebServlet("/xxx")` 注解映射路径。所有业务逻辑走 `doPost`（表单提交），只有列表查看走 `doGet`。处理完后用 `request.getRequestDispatcher("/WEB-INF/views/xxx.jsp").forward(request, response)` 转发到 JSP，或用 `response.sendRedirect()` 重定向。

#### 第 14 步：创建 `LoginServlet.java`
路径：`servlet/LoginServlet.java`
映射：`@WebServlet("/login")`

流程：
```
POST /login
    ↓
拿到 username、password 参数
    ↓
MD5(password) 加密
    ↓
UserDAO.findByUsernameAndPassword(username, encryptedPwd)
    ↓
查到了？ → session.setAttribute("user", user)
           → role=="admin" ? 重定向到 admin_home.jsp : user_home.jsp
没查到？ → request.setAttribute("error", "用户名或密码错误")
           → 转发回 login.jsp 显示错误
```

#### 第 15 步：创建 `RegisterServlet.java`
路径：`servlet/RegisterServlet.java`
映射：`@WebServlet("/register")`

流程：
```
POST /register
    ↓
校验：用户名不能为空、密码不能为空、两次密码一致
    ↓
UserDAO.findByUsername(username) → 不重复
    ↓
MD5 加密密码 → UserDAO.insert(user)
    ↓
成功 → 重定向到 login.jsp（提示注册成功）
失败 → 转发回 register.jsp 显示错误
```

#### 第 16 步：创建 `LogoutServlet.java`
路径：`servlet/LogoutServlet.java`
映射：`@WebServlet("/logout")`

流程：`session.invalidate()` → 重定向到 `login.jsp`。

#### 第 17 步：创建 `BookServlet.java`（最核心的Servlet）
路径：`servlet/BookServlet.java`
映射：`@WebServlet("/book/*")`

用 `action` 参数区分操作，统一入口：

| URL 模式 | method | action 参数 | 功能 |
|---|---|---|---|
| `/book/list` | GET | — | 图书列表 + 搜索 |
| `/book/detail` | GET | `&id=1` | 图书详情 |
| `/book/add` | GET | — | 显示添加页面（管理员） |
| `/book/add` | POST | — | 执行添加（管理员） |
| `/book/edit` | GET | `&id=1` | 显示编辑页面（管理员） |
| `/book/edit` | POST | — | 执行更新（管理员） |
| `/book/delete` | GET | `&id=1` | 删除图书（管理员） |

在 `doGet` 中，通过 `request.getPathInfo()` 获取路径后缀（如 `/list`），判断后执行对应逻辑。

#### 第 18 步：创建 `BorrowServlet.java`
路径：`servlet/BorrowServlet.java`
映射：`@WebServlet("/borrow/*")`

| URL 模式 | 功能 |
|---|---|
| `/borrow/do` → POST | 借书：创建 borrow_record，book.available_copies -1 |
| `/borrow/return` → POST | 还书：更新 return_date + status，book.available_copies +1 |
| `/borrow/list` → GET | 查看借阅记录（管理员看全部，用户看自己的）|

#### 第 19 步：创建 `UserServlet.java`

| URL 模式 | 功能 |

### 阶段七：View 层 — JSP 页面（第 20 步 ~ 第 31 步）

#### 前置原则
1. 所有 JSP 页面头部加 `<%@ page contentType="text/html;charset=UTF-8" language="java" %>`
2. 引入 JSTL 核心标签库：`<%@ taglib prefix="c" uri="jakarta.tags.core" %>`
3. 用 EL 表达式取值：`${sessionScope.user.username}`、`${bookList}`
4. 公共头部/尾部用 `<%@ include file="header.jsp" %>` 复用

#### 第 20 步：创建 `style.css`
路径：`webapp/css/style.css`

要点：设置统一的中文字体（微软雅黑）、浅灰背景、居中容器、表格样式、按钮样式、导航栏样式。不多于 150 行。

#### 第 21 步：创建 `header.jsp`
路径：`WEB-INF/views/header.jsp`

布局：顶部导航栏，包含"图书管理系统"标题 + 右侧用户名/退出按钮。用 `${sessionScope.user.role}` 判断显示管理员菜单还是用户菜单。

#### 第 22 步：创建 `footer.jsp`
路径：`WEB-INF/views/footer.jsp`

布局：底部版权信息，`<p>© 2026 图书管理系统</p>`。

#### 第 23 步：改写 `index.jsp`

内容：欢迎页，显示"欢迎使用图书管理系统"，提供"登录"和"注册"按钮。如果已登录则跳转到对应首页。

#### 第 24 步：创建 `login.jsp`

## 五、关键约定和坑位提醒

### 5.1 包名约定
所有 Java 类的 package 声明为 `com.qingshan.librarysystem.xxx`。例如：
- `package com.qingshan.librarysystem.model;`
- `package com.qingshan.librarysystem.dao;`

### 5.2 类名和文件命名
- Model：`User.java`、`Book.java`、`BorrowRecord.java`
- DAO：`UserDAO.java`、`BookDAO.java`、`BorrowRecordDAO.java`
- Servlet：`LoginServlet.java`、`BookServlet.java` 等
- JSP：小写+下划线，如 `book_list.jsp`、`admin_home.jsp`

### 5.3 常见坑
1. **中文乱码**：三处都要设 UTF-8 → 数据库 URL 参数 `characterEncoding=utf8`、Servlet 中 `request.setCharacterEncoding("UTF-8")`、JSP 头部 `contentType="text/html;charset=UTF-8"`
2. **JDBC 驱动类名**：MySQL 8.3 用 `com.mysql.cj.jdbc.Driver`
3. **Jakarta vs Javax**：JDK 21 + Servlet 6.0 全部用 `jakarta.servlet.*`，**绝不用** `javax.servlet.*`
4. **JSTL URI**：Jakarta 版用 `jakarta.tags.core`，而不是旧的 `http://java.sun.com/jsp/jstl/core`
5. **Session 过期**：`<session-config><session-timeout>30</session-timeout></session-config>` 在 web.xml 配置
6. **SQL 注入**：全部用 `PreparedStatement` + 占位符 `?`，绝不用字符串拼接 SQL
7. **密码明文**：存储时必须经 MD5 加密，不允许明文存数据库
8. **BorrowRecord 的关联字段**：user_id → users.id，book_id → books.id，建表时设了外键但代码里也要手动校验
